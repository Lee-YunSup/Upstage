# Agent 기반 논문 구독 서비스 📑
### Upstage Document Parse 및 Solar pro 3 활용

AI 분야에서는 매일 수십 편의 새로운 논문이 쏟아진다.
나 역시 처음에는 다음과 같은 문제를 겪었다.

- 관심 분야에서 어떤 학회나 저널이 중요한지 알기 어렵다
- 수많은 논문 중에서 무엇을 읽어야 할지 판단하기 힘들다
- 영어 논문을 처음부터 끝까지 읽는 데 시간이 많이 든다
- "매일 한 편씩만이라도 꾸준히 읽자"는 목표를 지키기 어렵다  
  
결국 좋은 논문이 나와도 놓치거나, 논문 읽기가 부담이 되어 학습이 중단되는 문제가 발생한다.  

이 프로젝트는 특정 학회의 논문을 정리하여  
매일 정해진 시간에 요약본을 보내주는 구독형 Agent를 만드는 것을 목표로 하였다.
전체 구조는 에이전트(agent.py), 스트림릿 Ui(app.py), 그리고 실행(runner.py)을 분리하여 설계되었다.


## 어떤 기술이 사용되었을까?
#### - arXiv API
최신 논문을 자동으로 수집

#### - Streamlit
이메일, 학회, 시간 설정을 위한 웹 UI 구성

#### - Upstage Document Parse
PDF를 구조화된 텍스트로 변환

#### - Upstage Chat (solar-pro3)
학회 추천에는 reasoning_effort=low로 빠른 응답  
논문 요약에는 reasoning_effort=high로 깊이 있는 이해

#### - SMTP 기반 이메일 전송
요약과 원본 PDF를 자동 발송
